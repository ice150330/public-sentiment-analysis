## 1. 项目概述

### 1.1 项目背景

在信息爆炸的社交媒体时代，公众情绪（Public Sentiment）的传播速度与影响力呈指数级增长。一条微博、一个抖音视频、一篇知乎回答，都可能在短时间内引发大规模情绪共鸣，形成社会舆论热点。然而，目前缺乏一套能够**实时监控、智能分析、预警预测**的综合性舆情分析系统。

### 1.2 项目目标

构建一套覆盖六大主流平台（微博、抖音、今日头条、百度、B站、知乎）的公众情绪智能分析系统，实现：

| 目标层级 | 具体目标 | 衡量标准 |
|---------|---------|---------|
| 基础层 | 多平台数据实时采集 | 覆盖6大平台，每6小时增量更新 |
| 分析层 | 情感倾向智能识别 | 情感分类准确率 ≥ 85% |
| 洞察层 | 热点话题自动聚类 | 自动发现每日Top10热点 |
| 预测层 | 舆情趋势预警预测 | 提前3-7天预测热度走势 |
| 展示层 | 交互式可视化大屏 | 3秒内响应，支持实时刷新 |

### 1.3 核心创新点

1. **多源异构数据融合**：首次将文本（微博/知乎）、视频元数据（B站/抖音）、搜索趋势（百度）统一建模
2. **跨平台情绪传播分析**：追踪同一事件在不同平台的情绪演化差异
3. **可解释性AI**：使用SHAP值解释模型预测原因，让"黑盒"变"白盒"
4. **实时预警推送**：异常舆情自动检测并触发多渠道告警

### 3.2 采集引擎架构

```python
# 采集引擎核心设计 (伪代码描述)

class SpiderEngine:
    """
    多平台采集引擎
    职责: 调度各平台采集器、管理采集状态、处理异常重试
    """
    
    def __init__(self):
        self.platforms = {
            'weibo': WeiboSpider(),
            'douyin': DouyinSpider(),
            'toutiao': ToutiaoSpider(),
            'baidu': BaiduSpider(),
            'bilibili': BilibiliSpider(),
            'zhihu': ZhihuSpider(),
        }
        self.scheduler = APScheduler()  # 定时调度
        self.task_queue = CeleryQueue()  # 异步任务队列
        self.state_manager = StateManager()  # 采集状态管理
    
    def schedule_all(self):
        """为所有平台注册定时采集任务"""
        for name, spider in self.platforms.items():
            self.scheduler.add_job(
                spider.crawl,
                trigger='interval',
                hours=6,
                id=f'crawl_{name}',
                replace_existing=True
            )
    
    async def crawl_with_retry(self, platform, max_retry=3):
        """带重试机制的采集执行"""
        spider = self.platforms[platform]
        for attempt in range(max_retry):
            try:
                result = await spider.crawl()
                self.state_manager.record_success(platform, result)
                return result
            except AntiCrawlDetected:
                await self.handle_anti_crawl(platform, attempt)
            except Exception as e:
                self.state_manager.record_failure(platform, e, attempt)
                await asyncio.sleep(2 ** attempt)  # 指数退避
        raise MaxRetryExceeded(platform)
```

### 3.3 反爬策略设计

| 策略 | 实现方式 | 目的 |
|------|---------|------|
| 请求频率控制 | 随机间隔 0.5-3 秒 | 模拟人类行为 |
| User-Agent 池 | 轮换 20+ 个真实 UA | 避免单一标识 |
| Cookie 管理 | 定期更新登录态 | 维持平台认证 |
| IP 代理池 | 住宅代理轮换 | 分散请求来源 |
| 浏览器指纹 | Playwright 模拟 | 绕过 JS 检测 |
| 失败重试 | 指数退避 (1s, 2s, 4s) | 提高成功率 |

## 5. 分析引擎层设计

### 5.1 情感分析引擎

#### 5.1.1 模块架构

```
情感分析引擎
│
├─ 预处理模块
│   ├─ 文本清洗 (去URL、去@、去表情)
│   ├─ 分词 (jieba / LTP)
│   └─ 去停用词
│
├─ 特征提取模块
│   ├─ 传统特征: TF-IDF、TextRank关键词
│   └─ 深度学习特征: BERT 768维向量
│
├─ 分类模型
│   ├─ 基线模型: SVM + TF-IDF
│   ├─ 深度学习: TextCNN / BiLSTM
│   └─ 预训练模型: BERT/RoBERTa 微调
│
└─ 后处理模块
    ├─ 置信度校准
    ├─ 阈值动态调整
    └─ 多平台情感对齐
```

#### 5.1.2 模型对比设计

| 模型 | 架构 | 准确率(目标) | 推理速度 | 适用场景 |
|------|------|-----------|---------|---------|
| SVM + TF-IDF | 传统机器学习 | 75% | 极快 | 基线对比 |
| TextCNN | 卷积神经网络 | 80% | 快 | 轻量部署 |
| BiLSTM + Attention | 循环神经网络 | 82% | 中等 | 长文本分析 |
| BERT-base | 预训练Transformer | 88% | 较慢 | 主模型 |
| RoBERTa-wwm-ext | 优化预训练 | 90% | 较慢 | 最终方案 |

```python
# 预测引擎架构 (伪代码)

class TrendForecaster:
    """
    舆情趋势预测引擎
    输入: 话题历史热度时序数据
    输出: 未来3-7天热度预测 + 置信区间
    """
    
    def __init__(self):
        self.models = {
            'prophet': ProphetForecaster(),      # 基线: Facebook Prophet
            'arima': ARIMAForecaster(),         # 统计方法
            'lstm': LSTMForecaster(),            # 深度学习
            'transformer': TransformerForecaster()  # 最终方案
        }
    
    def ensemble_predict(self, topic_id, horizon=7):
        """
        集成预测: 多模型加权平均
        """
        predictions = {}
        for name, model in self.models.items():
            predictions[name] = model.predict(topic_id, horizon)
        
        # 加权融合 (权重基于历史验证集MAE)
        weights = self.calculate_weights()
        ensemble = self.weighted_average(predictions, weights)
        
        return {
            'point_forecast': ensemble['mean'],
            'confidence_interval': (ensemble['lower'], ensemble['upper']),
            'model_contributions': weights,
            'horizon_days': horizon
        }
```

```python
# SHAP 解释器设计

class SentimentExplainer:
    """
    情感分析可解释性模块
    使用 SHAP (SHapley Additive exPlanations) 解释模型预测
    """
    
    def explain(self, text, model, tokenizer):
        """
        生成单条文本预测的解释
        """
        explainer = shap.Explainer(model, tokenizer)
        shap_values = explainer([text])
        
        return {
            'prediction': shap_values[0].base_values + shap_values[0].values.sum(),
            'word_contributions': [
                {'token': token, 'shap_value': value, 'direction': 'positive' if value > 0 else 'negative'}
                for token, value in zip(shap_values[0].data, shap_values[0].values)
            ],
            'top_positive_words': self.get_top_words(shap_values, direction='positive', n=5),
            'top_negative_words': self.get_top_words(shap_values, direction='negative', n=5),
            'summary_plot': shap.plots.text(shap_values[0])  # 可视化
        }
```

```python
# Celery 任务定义

@celery_app.task(bind=True, max_retries=3)
def analyze_sentiment_batch(self, article_ids: List[int]):
    """
    批量情感分析异步任务
    """
    try:
        articles = Article.query.filter(Article.id.in_(article_ids)).all()
        results = []
        for article in articles:
            result = sentiment_engine.analyze(article.content)
            results.append({
                'source_id': article.id,
                'source_type': 'article',
                **result
            })
        
        # 批量写入数据库
        SentimentResult.bulk_insert(results)
        
        return {'status': 'success', 'processed': len(results)}
    
    except Exception as exc:
        raise self.retry(exc=exc, countdown=60)


@celery_app.task
def daily_clustering_task():
    """
    每日热点聚类定时任务
    由 Celery Beat 每天凌晨 2:00 触发
    """
    yesterday = datetime.now() - timedelta(days=1)
    articles = Article.query.filter(
        Article.crawl_time >= yesterday
    ).all()
    
    clusters = clustering_engine.cluster(articles)
    clustering_engine.save_results(clusters, date=yesterday.date())
    
    return {'status': 'success', 'clusters_found': len(clusters)}
```

## 11. 风险与应对

### 11.1 技术风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| 反爬升级导致采集失败 | 高 | 高 | 多账号轮换、代理池、降低频率、备用方案 |
| 模型准确率不达标(<85%) | 中 | 高 | 使用预训练模型微调、增加标注数据、模型集成 |
| 数据量不足影响训练 | 中 | 高 | 连续采集30天+、使用公开数据集补充 |
| 前端性能瓶颈(大数据量) | 低 | 中 | 后端分页、前端虚拟滚动、数据采样 |
| 并发量大导致服务崩溃 | 低 | 高 | 限流、缓存、异步处理、水平扩展 |

### 11.2 进度风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| 爬虫开发超时 | 中 | 高 | 优先实现核心平台(微博/知乎)，其他平台降级为可选 |
| 模型训练耗时超预期 | 中 | 高 | 使用预训练模型而非从头训练，缩短训练周期 |
| 前端图表复杂度高 | 低 | 中 | 复用ECharts示例，分阶段迭代优化 |
| 论文撰写时间不足 | 中 | 高 | 边开发边记录技术细节，预留2周纯写作时间 |

### 11.3 合规风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| 爬虫违反平台TOS | 低 | 高 | 仅用于学术研究、控制频率、不商用、匿名化处理 |
| 个人隐私数据泄露 | 低 | 极高 | 不存储用户ID/手机号、数据脱敏、访问控制 |
| 敏感舆情数据处理 | 中 | 高 | 系统声明仅用于研究、不传播原始数据、导师知情 |
