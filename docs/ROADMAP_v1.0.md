## 现状诊断

当前系统处于**"骨架完整、血肉不足"**的状态：

- ✅ 数据 pipeline 通了（6 平台采集、定时任务、去重）
- ✅ 监控大屏能看了（Dashboard 4 标签页、14 个图表）
- ⚠️ 智能分析全是模拟实现（聚类、传播、预测、解释）

- ❌ 无数据导出能力
- ❌ 无实时推送

## P0 安全基线与数据基础设施 | 预估 3-4 周

**目标**：让系统达到可对外发布的安全和可用基线。

### 后端

#### ① 用户认证体系
- JWT Token 认证中间件
- 用户表（管理员/分析师/访客三角色）
- 登录/注册/密码重置 API
- 接口权限装饰器（`@require_admin`, `@require_analyst`）

#### ② SQLite 数据库加固
- 保持现有 SQLite 存储形态
- 补充关键查询索引与连接配置检查
- 数据备份脚本 + 回滚方案

#### ③ 数据导出功能
- 热榜数据导出 Excel/CSV
- 情感分析报告导出 PDF（含图表）
- 预警事件导出带时间线

#### ④ API 安全加固
- 请求频率限制（Rate Limiting）
- SQL 注入防护审计
- 敏感字段脱敏（`raw_data` 不返回给非管理员）

### 前端

#### ⑤ 登录/注册页面
#### ⑥ 权限感知路由（无权限自动跳转）
#### ⑦ 用户个人中心（修改密码、查看操作日志）

### 验收标准
- 未登录无法访问任何数据页面
- 管理员可导出全部数据，分析师只能导出自己有权限的平台
- SQLite 热点查询在当前部署数据量下保持可接受响应，关键查询具备索引与备份方案

## P1 真实 ML 模型落地 | 预估 4-6 周

**目标**：把"摆样子"的智能分析替换成真正能运行的模型。

### 1. 情感分析模型（最高优先级）

**后端**：
- 接入轻量级中文情感模型（bert-base-chinese-sentiment 或 MacBERT）
- 模型服务化：独立进程/容器，通过 HTTP/gRPC 与主服务通信
- 支持批量推理（batch_size=32）
- 模型版本管理（A/B 测试、灰度发布）
- 低置信度样本人工复核队列

**前端**：
- 情感分析结果展示置信度条
- 低置信度标红提醒
- 单条情感分析可点击查看模型解释（关键词高亮）

### 2. 主题聚类模型

**后端**：
- 文本嵌入：sentence-transformers（paraphrase-multilingual-MiniLM-L12-v2）
- 聚类算法：HDBSCAN（自动确定聚类数）+ K-Means（固定聚类数）双策略
- 聚类结果持久化 + 增量更新
- 聚类关键词提取（TF-IDF / TextRank）

**前端**：
- 聚类结果页面：每个聚类的关键词云、话题列表、情感分布
- 聚类对比（不同时间窗口的聚类演变）
- 话题从属关系可视化（树形图）

### 3. 传播路径分析

**后端**：
- 跨平台话题相似度匹配（基于标题 embedding 的 cosine 相似度）
- 传播时序建模（话题首次出现平台 → 后续平台的时间差）
- 影响力传播模型（独立级联模型简化版）
- 传播路径图数据生成

**前端**：
- 传播路径桑基图（平台 → 话题 → 时间轴）
- 传播速度指标（跨平台传播耗时）
- 关键节点识别（引爆点标记）

### 验收标准
- 情感分析准确率 > 85%（在标注测试集上）
- 聚类结果人工评估：同一聚类内话题相关性 > 70%
- 传播路径能正确识别 80% 以上的跨平台相同话题

## P2 高级分析完善与体验优化 | 预估 3-4 周

**目标**：预测、解释、实时推送三大能力补齐。

### 1. 趋势预测模型

**后端**：
- 时序模型：Prophet 或 statsmodels ARIMA
- 特征工程：历史热度、情感变化率、平台扩散速度
- 预测结果置信区间
- 预测准确率自评估（滚动验证）

**前端**：
- 趋势预测图表（历史线 + 预测虚线 + 置信区间阴影）
- 预测预警（预测热度将突破阈值时提前告警）

**2026-07-13 进展**：已新增 SQLite 时序预测服务，支持 ARIMA 优先、EMA 回退、滚动验证、置信区间、预测信号与前端预测页展示，详见 `docs/P2_Trend_Forecast.md`。

### 2. 模型解释系统

**后端**：
- 接入 LIME / SHAP 解释框架
- 情感分析解释：哪些词推动了正面/负面判定
- 聚类解释：每个聚类的代表性文档
- 预测解释：哪些特征对预测结果贡献最大

**前端**：
- 情感分析结果页展示关键词贡献度条形图
- 聚类详情页展示代表性话题
- 预测结果页展示特征重要性排名

**2026-07-18 进展**：已落地 LIME 风格扰动解释服务（in-house、无重依赖）、聚类代表话题与预测特征贡献确认，详见 `docs/P2_Model_Explanation.md`。

### 3. 实时推送

**后端**：
- WebSocket 连接管理（每用户一个连接）
- 事件驱动：新预警、数据异常、采集完成
- 消息队列：Redis Pub/Sub 或 RabbitMQ

**前端**：
- 全局消息通知组件（右上角 toast）
- 预警实时弹窗
- 数据更新实时刷新（替代轮询）

### 验收标准
- 预测 MAPE（平均绝对百分比误差）< 20%
- 解释结果用户满意度 > 75%（抽样问卷）
- 实时消息延迟 < 3 秒

## P3 工程化与运维优化 | 预估 2-3 周

**目标**：爬虫稳定、性能可观测、部署自动化。

### 1. 全平台爬虫稳定
- 各平台独立 Cookie 池管理
- 代理 IP 池集成（支持按平台切换）
- 反爬策略：请求间隔抖动、User-Agent 轮换、浏览器指纹随机化
- 失败自动重试（指数退避）
- 采集成功率监控告警（低于 70% 自动通知）

### 2. 性能优化
- 数据库索引优化（热点查询 EXPLAIN ANALYZE）
- 缓存层：Redis 缓存热点数据（Overview、平台矩阵）
- 前端懒加载：大屏图表按需渲染
- 图片/资源 CDN 加速

### 3. 可观测性
- 结构化日志（JSON 格式，含 trace_id）
- Prometheus + Grafana 监控大盘
- 关键指标：API 延迟 P99、错误率、爬虫成功率、模型推理耗时
- 告警规则：错误率 > 5%、磁盘 > 80%、内存 > 85%

### 4. 部署自动化
- Docker Compose 全栈编排
- CI/CD：GitHub Actions 自动测试 + 构建 + 部署
- 数据库变更管理：Alembic migration 自动化
- 蓝绿部署（零停机更新）

### 验收标准
- 爬虫整体成功率 > 90%
- API P99 延迟 < 200ms
- 系统可用性 > 99.5%
